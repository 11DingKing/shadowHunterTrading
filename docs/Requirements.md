# Shadow-Hunter v10.4 需求规格说明书

## 1. 项目概述

### 1.1 项目名称
Shadow-Hunter v10.4 (影子猎手)

### 1.2 项目背景
基于"方块兽"游戏地狱板块的自动化博弈辅助软件，采用多层算法探针与 AI 人格投票机制进行决策。

### 1.3 技术栈
- **语言**: Python 3.11+
- **GUI框架**: PyQt6
- **容器化**: Docker + noVNC (Web 访问桌面应用)

---

## 2. 核心功能需求

### 2.1 数据结构与配置 (config.py)

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| YAO_MAP | Dict | {1:熊妖, 2:牛妖, 3:狐妖, 4:蛇妖, 5:蛛妖, 6:兔妖, 7:象妖, 8:猴妖} | 8妖位置映射 |
| BASE_PROB | Float | 0.125 | 基础概率常量 |
| sequence | List | [100, 600, 1300, 2700] | 投注序列 |
| snow_limit | Int | 2 | 滚雪球轮次上限 |
| shadow_unlock_wins | Int | 3 | 影子模式解锁连胜数 |
| shadow_unlock_rate | Float | 0.8 | 解锁胜率阈值 |
| soft_landing | Bool | True | 是否开启降级复投 |

### 2.2 数学探针算法 (core/algorithms.py)

Layer 1 共 9 种探针算法：

| ID | 算法名 | 逻辑描述 |
|----|--------|----------|
| 1 | algo_trend | 统计最近50期出现最多的数字（追热） |
| 2 | algo_gap | 统计最近80期出现最少的数字（补冷） |
| 3 | algo_zscore | 返回最近20期出现频率最高的 |
| 4 | algo_mirror | 返回上一期结果 |
| 5 | algo_anti_double | 若连续两期相同，返回该数字；否则随机 |
| 6 | algo_time | 返回 (当前时间戳秒数 % 8) + 1 |
| 7 | algo_entropy | 随机返回 1-8 |
| 8 | algo_markov | 随机返回 1-8 |
| 9 | algo_shadow | 随机返回 1-8 |

### 2.3 AI 人格系统 (core/agents.py)

三大 AI 人格，各负责 3 个算法：

| 人格名 | 负责算法 | 投票规则 |
|--------|----------|----------|
| Deep | 算法 1,2,3 | 2/3 一致则投票，否则弃权 |
| Tongyi | 算法 4,5,6 | 2/3 一致则投票，否则弃权 |
| Doubao | 算法 7,8,9 | 2/3 一致则投票，否则弃权 |

**EMA 权重更新公式**: `new_score = old_score * 0.8 + (1 if win else -1) * 0.2`

### 2.4 仲裁系统 (core/arbiter.py)

- 收集所有有效投票
- 按 EMA 权重加权统计
- 返回得分最高的位置作为最终建议
- 全部弃权时返回 None

### 2.5 策略引擎 (core/strategy.py)

**状态机**:
```
WARMUP (预热10轮) → REAL (实盘) ⇄ SHADOW (影子防御)
```

**资金链逻辑**:
- 序列投注: 按 sequence 数组递进
- 滚雪球: 赢则累积本金+盈利，达到 snow_limit 止盈
- 输则进入 SHADOW 模式

**影子防御**:
- 记录虚拟胜负
- 连胜达标后解锁回 REAL
- 解锁时开启 soft_landing (金额减半)

### 2.6 用户界面 (ui/)

**风格**: 深色地狱风格
- 背景色: #1e1e1e
- 高亮色: #00bcd4 (青色), #ff3d00 (熔岩红)

**三页面布局**:

| 页面 | 功能 |
|------|------|
| 监控台 | 状态显示、余额、3个AI人格卡片、8妖按钮网格 |
| 配置 | 序列输入、滚雪球次数、解锁参数 |
| 数据 | 历史记录表格 (时间/神仙猿/躲避/结果/状态/矿石) |

### 2.7 主程序 (main.py)

- 整合所有模块
- 模拟游戏循环线程
- 反侦察延迟: `time.sleep(random.uniform(1.5, 4.0))`
- 信号机制更新 UI

## 3. Docker 交付标准

### 3.1 适配方案
采用 noVNC 容器化策略，将 PyQt6 桌面应用通过 Web 浏览器访问。

### 3.2 访问方式
```bash
docker compose up --build -d
# 访问: http://localhost:8081
```

### 3.3 跨平台支持
- ARM64 (Apple Silicon) ✅
- AMD64 (x86) ✅
